# Design to 10.0 on the CodeScene delta

**When:** writing new code or refactoring under the CodeScene delta gate.

**Principle:** design every new function to score 10.0 on the PR delta from the
first push. Never inherit low-CC code into a small PR — either split the
refactor into its own PR or suppress the deltas in CodeScene's UI before
opening.

**Why:** the OpenADT CodeScene gate uses a "pay-down-tech-debt" profile. The
delta is **absolute** per file, not per-method. A file that was 8.0 before the
PR and remains 8.0 after will still fail if any new code was added (the delta
sees the whole file, not just the new lines).

**Heuristics:**
- Target function CC ≤ 6 (hard cap 9). The first `&&` adds 1.
- Group args into a context/options object at 4+.
- Extract method at 2+ cohesive blocks at nesting depth ≥ 2.
- Extract predicate at 2+ logical operators in one expression.
- Use `test.each` for similar-shape test assertions.
- If a function does 3+ things, extract helpers.

**Source:** PR #62 `/act` session (2026-06-07). The first refactor agent claimed
success but didn't actually reduce complexity — the live `gh pr checks` was the
only valid evidence. A second, deeper refactor was needed.
